What is TeaVM?

TeaVM is an ahead-of-time (AOT) compiler that translates Java bytecode into JavaScript or WebAssembly (aka Wasm). This is a key difference from GWT, which transpiles Java source code directly into JavaScript.

Because TeaVM works at the bytecode level, it can compile not only Java but any language that targets the JVM â€” such as Kotlin and Scala â€” making it a versatile choice for modern web development.

Why TeaVM support matters

  • WebAssembly performance

One of the biggest benefits of TeaVM is its WebAssembly output (in addition to JS). Unlike JavaScript, WebAssembly is a newer low-level binary format â€” already compiled before it reaches the browser â€” allowing near-native performance and now supported by all modern browsers. With TeaVM, WebFX applications can now compile to Wasm and run faster â€” while still offering direct access to the browser DOM, something most other Wasm toolchains cannot do.

  • Kotlin and Scala support

TeaVMâ€™s bytecode-based design means it supports all JVM languages â€” including Kotlin and Scala â€” right out of the box. This expands WebFX beyond Java, allowing teams to choose their preferred language without giving up the familiar JavaFX programming model.

  • Future-proof addition

By integrating TeaVM alongside GWT, WebFX becomes more future-proof. Developers can pick between JavaScript and WebAssembly, ensuring long-term flexibility as the web platform continues to evolve.

How we integrated TeaVM into WebFX

Adding TeaVM support in addition to GWT turned out to be much easier than expected â€” thanks to the WebFX Platform, our foundation layer for Java-based cross-platform development (desktop, mobile, and web).

While WebFX focuses on bringing JavaFX applications to the web, the WebFX Platform is more general-purpose and not tied to JavaFX. It provides the modular building blocks that make WebFX possible â€” including APIs for UI, I/O, and browser integration.

To make TeaVM compatible with the WebFX stack - which mainly relies on Elemental2 for the web platform â€” we created a polyfill for Elemental2, this Google library that exposes browser APIs (like the DOM, events, and storage) to Java code. This polyfill is now part of the WebFX Platform and allows TeaVM to seamlessly understand and compile the same Elemental2-based code used by GWT.

At the moment, the polyfill covers only the subset of the Elemental2 API used by WebFX itself. However, it can potentially be expanded to cover any additional Elemental2 APIs. Increasing the coverage is a straightforward (if somewhat boring) task, so in principle, any Elemental2-based project could be compiled to Wasm through the WebFX Platform.

And it works! You can now build the same WebFX codebase with either GWT or TeaVM â€” with no changes to your application code.
