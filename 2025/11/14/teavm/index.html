WebFX now supports TeaVM: bringing WebAssembly and Kotlin to JavaFX on the Web!

Nov 14, 2025

We‚Äôre excited to announce that WebFX now officially supports TeaVM, a Java-to-WebAssembly compiler ‚Äî unlocking faster startup times and broader JVM language compatibility, including Kotlin and Scala.

What is TeaVM?

TeaVM is a compiler that translates Java bytecode into JavaScript or WebAssembly. This is a key difference from GWT, which transpiles Java source code directly into JavaScript.

Because TeaVM works at the bytecode level, it can compile not only Java but any language that targets the JVM ‚Äî making it a versatile choice for modern web development.

Why TeaVM support matters

  • WebAssembly performance

Unlike JavaScript, WebAssembly (aka Wasm) is a newer low-level binary format ‚Äî already compiled before it reaches the browser ‚Äî allowing near-native performance and now supported by all modern browsers.

  • Direct access to the browser DOM

This is something that other Wasm toolchains don‚Äôt offer because only JS can access to the DOM in reality, but TeaVM made its magic with some glue between Wasm and JS. Not only does it work, but it‚Äôs completely transparent to the developer. A real game-changer!

  • Kotlin (and beyond)

TeaVM‚Äôs bytecode-based design means it supports all JVM languages ‚Äî including Kotlin and Scala ‚Äî right out of the box. This expands WebFX beyond Java, allowing teams to choose their preferred language without giving up the JavaFX programming model.

  • Future-proofing WebFX

The reason WebAssembly is gaining attraction is that its bytecode is language-neutral by design (unlike Java bytecode). This opens the door for many other languages to compile to Wasm and run efficiently in the browser. While JS won‚Äôt disappear, it‚Äôs likely that more and more web apps will be written in other languages and compiled to Wasm for better performance and safety.

By supporting Wasm today, WebFX is already aligned with this shift, positioning it as part of the emerging ‚Äúpost-JavaScript‚Äù era while remaining fully compatible with JS.

How we integrated TeaVM into WebFX

Adding TeaVM support in addition to GWT turned out to be much easier than expected ‚Äî thanks to the WebFX Platform, our foundation layer for Java-based cross-platform development (desktop, mobile, and web). While WebFX focuses on bringing JavaFX to the web, the WebFX Platform is more general-purpose and not tied to JavaFX. It provides the modular building blocks that make WebFX possible.

To make TeaVM compatible with the existing WebFX code - whose browser interaction is written with Elemental2, the Google library that exposes browser APIs (DOM, events, storage, etc‚Ä¶) to Java ‚Äî we simply created a polyfill of Elemental2 for TeaVM. This polyfill is now part of the WebFX Platform and allows TeaVM to seamlessly understand and compile the same Elemental2-based code used by GWT.

This polyfill covers only the subset of the Elemental2 API used by WebFX. However, it can potentially be expanded to cover any additional Elemental2 APIs. Increasing the coverage is a straightforward (if somewhat boring) task, so in principle, it could compile any Elemental2-based project to Wasm.

And it works! You can now build the same WebFX codebase with either GWT or TeaVM ‚Äî with no changes at all to your application code.

What‚Äôs next?

To ensure compatibility with older browsers, we plan to support dual TeaVM/GWT builds. The WebFX tooling will automatically select the appropriate version at runtime, giving developers the best of both worlds ‚Äî modern WebAssembly performance where possible, and full JS backward compatibility where needed.
