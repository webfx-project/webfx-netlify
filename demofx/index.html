New demo!

Sep 25, 2022

You may haven’t noticed it because we are writing this post only now, but we published a new demo on the WebFX website about a month ago.

This demo is based on the excellent DemoFX library by Chris Newman. It’s a JavaFX canvas library that offers a collection of effects that were primarily designed for JavaFX canvas benchmarks, but that can be used and chained to make animations too. We made a partial port of this library in WebFX Libs, our dedicated space for WebFX libraries.

We first made this DemoFX kitchen sink to test and visualize each effect individually, and finally we chose to sequence them in a particular order, with the music of David Newman played in the background (also provided by the DemoFX library). We selected specific transition times to match the musical rhythm. The demo starts with an introduction animation asking to click to play the actual demo, because in the browser sandbox, it’s not possible to start the music without a user interaction first. This resulted in this single java source file.

Most animations should be fluid, except maybe the fractal rings (2nd sequence) on low devices or big screens, especially when another animation is running simultaneously (such as the fading star field or the Sierpinski effect). You may try the native version to get better results. But on modern devices (we tested on an iPad mini 6), the web performance is already excellent.

This WebFX demo mainly focuses on the canvas support and the MediaPlayer sound support (including the spectrum analyser as demonstrated by the equalizer). It also shows how WebFX can produce lightweight and performant web apps (the JS is 128kB, and it loads and starts quickly).
