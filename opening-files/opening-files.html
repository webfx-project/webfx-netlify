Accessing local files with WebFX

Oct 18, 2022

First of all, a big thank you to the whole JavaFX community for warmly welcoming us on Twitter. In just a couple of days after creating our account, you were already 100+ to follow us! And JFX-Central kindly listed us in their Links of the Week on September 30, including our last blog and demo. Thank you all for your interest in WebFX! ❤️

Thank you also for your questions. And it appeared that the most frequent question was about accessing local files. So we investigated (as this feature was not yet implemented), and we are glad to announce that WebFX can now access local files, as demonstrated with our brand-new Files demo.

This demo lets you select local files with either the Choose files button or drag & drop. The files are listed with their icon and other simple details, and a thumbnail is displayed for images. A grid view is also available with just the thumbnails. You can open your images in full size, and navigate using left & right keys or mouse clicks or swipes on touch screens. Same with your audio files, which are played together with the equaliser from our DemoFX library. Video files will be played very soon (only the audio is supported for now). Text files can be displayed in a TextArea.

Facing the issues with the JavaFX FileChooser and Java File API

Our first investigation was rather bad news: it appeared that it was impossible to emulate the JavaFX FileChooser API in the browser, mainly for the 2 following reasons:

  • In JavaFX, opening the file chooser is done programmatically through a call to FileChooser.showOpenDialog(), which in addition is a blocking call - something prohibited in JavaScript - while in the browser it is done through a user interaction with the HTML input element <input type="file"> which is rendered by default like this:

  • The Web File object returned by the browser file chooser has very limited information about the actual local file. For example, it doesn’t let you know its location on the local machine, and its URL is a kind of random UUID that is valid only for the time of the session. With such a limited sandbox object, it’s impossible to emulate java.io.File, the object returned by the JavaFX FileChooser.

The WebFX solution: an alternative cross-platform API

While it’s impossible to emulate the JavaFX FileChooser and the Java File API in the browser, the opposite is possible: it’s quite simple to emulate the Web FilePicker and Web File API in Java/JavaFX. This is therefore the WebFX approach.

In our WebFX Platform, you will see these 3 new modules:

  1. webfx-platform-file : a WebFX cross-platform File API
  2. webfx-platform-file-java : its implementation in Java (based on Java File)
  3. webfx-platform-file-gwt : its implementation in GWT (based on Web File)

And in our WebFX Extras library, these 3 new modules:

  1. webfx-extras-filepicker : a WebFX cross-platform FilePicker API
  2. webfx-extras-filepicker-openjfx : its implementation in OpenJFX (based on FileChooser)
  3. webfx-extras-filepicker-gwt : its implementation in GWT (based on Web FilePicker)
